# Вимкнути підтвердження email при реєстрації

Якщо при вході з’являється **«Email not confirmed»** або **«Будь ласка, підтвердіть email»**, у Supabase увімкнено підтвердження пошти. Його потрібно вимкнути вручну.

> **Це не SQL.** Не вставляйте цей файл у SQL Editor. Виконайте кроки нижче в інтерфейсі Supabase.

---

### Кроки

1. Відкрийте [Supabase Dashboard](https://supabase.com/dashboard) → оберіть **ваш проект**.
2. У лівому меню: **Authentication** → **Providers** (саме **Providers**, не «Emails»).
3. У списку провайдерів (Google, Email, GitHub тощо) натисніть **Email**.
4. На сторінці налаштувань Email знайдіть **«Confirm email»** і **вимкніть** перемикач.
5. Натисніть **Save** / **Зберегти**.

**Не плутайте:**  
- **Emails** (Templates, SMTP) — шаблони листів, не те.  
- **Providers → Email** — тут перемикач «Confirm email».

Після цього нові користувачі зможуть увійти одразу після реєстрації, без листа з посиланням. Існуючі непідтверджені акаунти теж зможуть увійти.

---

**Також виправлено в додатку:**
- **406 (Not Acceptable):** усі звернення до `profiles` по `id` використовують `.maybeSingle()` замість `.single()`, щоб не було помилки, коли профілю ще немає.
- **400 (Bad Request) при створенні профілю:** замість `upsert` використовується `ensureProfile` (insert, при дублікаті — update), щоб уникнути 400.
